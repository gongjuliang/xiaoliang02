# NATT 内网穿透使用文档

本文档说明如何启动、配置和使用 `nattserver` 服务端与 `nattuser` 客户端。当前版本首版能力为 TCP 端口穿透，服务端与客户端通信采用控制连接和数据连接分离模型。

```text
  nattuser\          客户端程序
  commonGin\         参考项目

```

```text

```

## 2. 默认端口和账号

服务端默认端口：

| 用途 | 默认值 |
| --- | --- |
| Web 管理后台 | `0.0.0.0:8080` |
| 客户端控制连接 | `0.0.0.0:7000` |
| 客户端数据连接 | `0.0.0.0:7001` |
| 公网映射端口范围 | `10000-60000` |
| MCP 服务 | 默认关闭，开启后 `127.0.0.1:8081` |

客户端默认端口：

| 用途 | 默认值 |
| --- | --- |
| 本地 Web 管理后台 | `127.0.0.1:18080` |
| MCP 服务 | 默认关闭，开启后 `127.0.0.1:18081` |

默认管理员账号：

| 项目 | 用户名 | 密码 |
| --- | --- | --- |
| 服务端 | `admin` | `admin123456` |
| 客户端 | `admin` | `admin123456` |

首次初始化数据库时会创建默认管理员。生产环境建议在第一次启动前设置环境变量：

```powershell
$env:NATT_SERVER_ADMIN_USERNAME="admin"
$env:NATT_SERVER_ADMIN_PASSWORD="change-this-server-password"
$env:NATT_USER_ADMIN_USERNAME="admin"
$env:NATT_USER_ADMIN_PASSWORD="change-this-client-password"
```

如果数据库已经初始化，修改这些环境变量不会自动改已有用户密码。

## 4. 基本使用流程

### 4.1 服务端创建客户端授权

1. 打开服务端后台：`http://127.0.0.1:8080`
2. 使用管理员账号登录。
3. 进入“客户端”页面。
4. 点击“新增客户端”。
5. 填写客户端名称并保存。
6. 保存后会弹出客户端秘钥，只显示这一次，请记录下来。

客户端秘钥会以哈希形式存储在服务端数据库中，后续只能轮换，不能查看原文。

### 4.2 客户端连接服务端

1. 打开客户端后台：`http://127.0.0.1:18080`
2. 使用管理员账号登录。
3. 进入“服务端”页面。
4. 新增服务端连接：
   - 服务端地址：公网服务端 IP 或域名
   - 控制端口：默认 `7000`
   - 数据端口：默认 `7001`
   - 客户端秘钥：填写服务端创建客户端时生成的秘钥
   - TLS：按服务端配置选择
   - 自启动：建议开启
5. 点击连接或保存后等待状态变为 connected。

### 4.3 创建 TCP 隧道

在服务端后台进入“隧道”页面，新增隧道：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 名称 | `ssh-home` | 隧道名称 |
| 客户端 | `client-a` | 已授权并在线的客户端 |
| 本地地址 | `127.0.0.1` | 客户端机器上可访问的内网服务地址 |
| 本地端口 | `22` | 客户端机器上的内网服务端口 |
| 公网监听地址 | `0.0.0.0` | 服务端监听地址 |
| 公网监听端口 | `10022` | 外部访问端口，需在允许范围内且不能冲突 |
| 自启动 | 按需 | 客户端重连后自动恢复 |

保存后点击启动。启动成功后，外部访问：

```text
服务端公网IP:公网监听端口
```

会被转发到：

```text
客户端本地地址:客户端本地端口
```

示例：

```text
1.2.3.4:10022 -> 客户端 127.0.0.1:22
```

## 5. 配置说明

### 5.1 服务端配置

文件：`D:\2025Code\ai\natt\nattserver\config\config.json`

关键配置：

```json
{
  "http": {
    "host": "0.0.0.0",
    "port": 8080
  },
  "database": {
    "path": "data/nattserver.db"
  },
  "log": {
    "dir": "logs",
    "level": "info"
  },
  "auth": {
    "jwt_secret": "change-me-nattserver-dev-secret"
  },
  "protocol": {
    "control_host": "0.0.0.0",
    "control_port": 7000,
    "data_host": "0.0.0.0",
    "data_port": 7001,
    "tls": {
      "enabled": false,
      "cert_file": "",
      "key_file": ""
    }
  },
  "tunnel": {
    "remote_port_min": 10000,
    "remote_port_max": 60000
  }
}
```

生产环境至少建议修改：

- `auth.jwt_secret`
- 管理员默认密码
- `protocol.tls.enabled`
- 防火墙放行 Web、控制、数据和映射端口

### 5.2 客户端配置

文件：`D:\2025Code\ai\natt\nattuser\config\config.json`

关键配置：

```json
{
  "http": {
    "host": "127.0.0.1",
    "port": 18080
  },
  "database": {
    "path": "data/nattuser.db"
  },
  "log": {
    "dir": "logs",
    "level": "info"
  },
  "auth": {
    "jwt_secret": "change-me-nattuser-dev-secret"
  },
  "server_defaults": {
    "server_host": "127.0.0.1",
    "control_port": 7000,
    "data_port": 7001,
    "use_tls": false
  }
}
```

生产环境至少建议修改：

- `auth.jwt_secret`
- 管理员默认密码
- `server_defaults.server_host`
- 按服务端 TLS 配置同步 `server_defaults.use_tls`

### 5.3 JSON 与 YAML 兼容策略

当前默认配置格式为 JSON，默认路径为 `config/config.json`。启动时不传 `-config` 时，程序会优先读取 `config/config.json`；如果该文件不存在，会自动回退读取旧的 `config/config.yaml`。

已有 YAML 部署可以继续显式使用：

```powershell
go run . -config config/config.yaml
```

迁移到 JSON 时，可以把原 YAML 字段按同名字段转换为 JSON。JSON 标准不支持注释，如果需要保留说明文字，请写在使用文档或单独的部署备注中，不要写进 `config.json`。

## 6. TLS 配置

当前控制连接和数据连接都支持可配置 TLS。

服务端启用示例：

```json
{
  "protocol": {
    "tls": {
      "enabled": true,
      "cert_file": "data/server.crt",
      "key_file": "data/server.key"
    }
  }
}
```

客户端连接服务端时需要将 `use_tls` 设为 `true`。开发环境可以关闭 TLS，生产环境建议开启。

## 7. MCP 使用

MCP 默认关闭。开启时必须配置访问令牌。

### 7.1 服务端 MCP

服务端配置：

```json
{
  "mcp": {
    "enabled": true,
    "host": "127.0.0.1",
    "port": 8081,
    "access_token": "replace-with-server-mcp-token"
  }
}
```

请求地址：

```text
POST http://127.0.0.1:8081/tools/call
Authorization: Bearer replace-with-server-mcp-token
Content-Type: application/json
```

请求格式：

```json
{
  "tool": "server.get_dashboard",
  "params": {}
}
```

服务端工具：

| 工具 | 说明 |
| --- | --- |
| `server.list_clients` | 查询客户端列表 |
| `server.get_client` | 查询单个客户端 |
| `server.list_tunnels` | 查询隧道列表 |
| `server.create_tunnel` | 创建隧道 |
| `server.start_tunnel` | 启动隧道 |
| `server.stop_tunnel` | 停止隧道 |
| `server.delete_tunnel` | 删除隧道 |
| `server.get_dashboard` | 查询仪表盘统计 |

示例：

```powershell
curl.exe -X POST http://127.0.0.1:8081/tools/call `
  -H "Authorization: Bearer replace-with-server-mcp-token" `
  -H "Content-Type: application/json" `
  -d "{\"tool\":\"server.list_clients\",\"params\":{\"page\":1,\"page_size\":20}}"
```

创建隧道示例：

```json
{
  "tool": "server.create_tunnel",
  "params": {
    "name": "ssh-home",
    "client_id": 1,
    "protocol": "tcp",
    "local_host": "127.0.0.1",
    "local_port": 22,
    "remote_host": "0.0.0.0",
    "remote_port": 10022,
    "auto_start": true,
    "remark": "ssh tunnel"
  }
}
```

### 7.2 客户端 MCP

客户端配置：

```json
{
  "mcp": {
    "enabled": true,
    "host": "127.0.0.1",
    "port": 18081,
    "access_token": "replace-with-client-mcp-token"
  }
}
```

请求地址：

```text
POST http://127.0.0.1:18081/tools/call
Authorization: Bearer replace-with-client-mcp-token
Content-Type: application/json
```

客户端工具：

| 工具 | 说明 |
| --- | --- |
| `client.list_servers` | 查询服务端连接配置 |
| `client.connect_server` | 连接指定服务端 |
| `client.disconnect_server` | 断开指定服务端 |
| `client.list_tunnels` | 查询本地可见隧道状态 |
| `client.get_network_status` | 查询本机网络状态 |

示例：

```powershell
curl.exe -X POST http://127.0.0.1:18081/tools/call `
  -H "Authorization: Bearer replace-with-client-mcp-token" `
  -H "Content-Type: application/json" `
  -d "{\"tool\":\"client.get_network_status\",\"params\":{}}"
```

MCP 返回格式：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

修改类 MCP 操作会写入审计日志。

- `debug`
- `info`
- `error`

## 11. 常见问题

### 11.1 登录失败

检查：

- 用户名和密码是否正确。
- 数据库是否已经初始化过旧密码。
- 是否触发登录限流。
- 生产环境下登录密码传输要求更严格，开发环境允许明文登录。

### 11.2 客户端无法连接服务端

检查：

- 服务端 `7000` 控制端口是否开放。
- 客户端填写的服务端地址、控制端口、数据端口是否正确。
- 客户端秘钥是否复制完整。
- 服务端是否禁用了该客户端。
- TLS 开关是否两边一致。

### 11.3 隧道启动失败

检查：

- 客户端是否在线。
- `remote_port` 是否在允许范围内。
- `remote_port` 是否已被占用。
- 客户端本地服务地址和端口是否可访问。
- 服务端防火墙是否放行公网映射端口。

### 11.4 公网端口无法访问

检查：

- 隧道状态是否为 running。
- 服务端公网 IP 是否正确。
- 云服务器安全组是否放行映射端口。
- 本地服务是否只监听了不可访问的地址。
- 客户端到服务端的数据连接端口 `7001` 是否可达。

### 11.5 MCP 返回 unauthorized

检查：

- `mcp.enabled` 是否为 `true`。
- `mcp.access_token` 是否配置。
- 请求头是否为 `Authorization: Bearer <token>`。

## 12. 生产环境建议

- 修改默认管理员密码。
- 修改 `auth.jwt_secret`。
- 开启控制连接和数据连接 TLS。
- MCP 仅绑定内网地址或本机地址。
- 使用强随机 MCP token。
- 限制服务端 Web 管理后台访问来源。
- 只开放必要公网映射端口。
- 定期备份 SQLite 数据库文件。
