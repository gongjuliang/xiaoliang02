# 小良内网穿透使用文档

本文说明如何启动、初始化、配置和使用 `nattserver` 服务端与 `nattuser` 客户端。当前版本只支持 TCP 端口穿透，通信模型为“控制连接 + 数据连接分离”。

前端品牌名：

- 服务端：`小良内网穿透-服务端`
- 客户端：`小良内网穿透-客户端`

## 1. 项目目录

```text
D:\2025Code\ai\natt
  nattserver\    服务端
  nattuser\      客户端
  skills\        项目 AI Skill
```

服务端默认文件：

```text
nattserver\xiaoliang02_server\config\config.json
nattserver\xiaoliang02_server\data\nattserver.db
nattserver\xiaoliang02_server\data\sm2_private.pem
nattserver\xiaoliang02_server\data\sm2_public.pem
nattserver\xiaoliang02_server\ssl\web.crt
nattserver\xiaoliang02_server\ssl\web.key
nattserver\xiaoliang02_server\logs\YYYY-MM-DD.log
nattserver\xiaoliang02_server\logs\audit\YYYY-MM-DD.jsonl
```

客户端默认文件：

```text
nattuser\xiaoliang02_user\config\config.json
nattuser\xiaoliang02_user\data\nattuser.db
nattuser\xiaoliang02_user\data\sm2_private.pem
nattuser\xiaoliang02_user\data\sm2_public.pem
nattuser\xiaoliang02_user\ssl\web.crt
nattuser\xiaoliang02_user\ssl\web.key
nattuser\xiaoliang02_user\logs\YYYY-MM-DD.log
nattuser\xiaoliang02_user\logs\audit\YYYY-MM-DD.jsonl
```

默认启动只读取上面的新母目录路径，不会自动读取旧的 `config/config.json`。如果显式传入 `-config`，程序按传入路径读取；`.json`、`.yaml`、`.yml` 都支持。

## 2. 默认端口

服务端：

| 用途 | 默认值 |
| --- | --- |
| Web 管理后台 | `0.0.0.0:25510` |
| MCP 接口 | `0.0.0.0:25510/mcp` |
| 客户端控制连接 | `0.0.0.0:25511` |
| 客户端数据连接 | `0.0.0.0:25512` |
| 公网映射端口范围 | `0-65535` |

客户端：

| 用途 | 默认值 |
| --- | --- |
| 本地 Web 管理后台 | `127.0.0.1:25520` |
| MCP 接口 | `127.0.0.1:25520/mcp` |
| 默认控制端口 | `25511` |
| 默认数据端口 | `25512` |

启动前会检查本机需要监听的端口。服务端检查 Web、控制、数据端口；客户端只检查本机 Web 端口。端口被占用时，会输出类似 `25510端口被占用` 的中文错误。

## 3. 首次初始化

默认启动时，如果 `xiaoliang02_server/config/config.json` 或 `xiaoliang02_user/config/config.json` 不存在，或数据库不存在、`users` 表为空，程序会进入初始化页面。

服务端初始化页：

```text
http://127.0.0.1:25510/init.html
```

客户端初始化页：

```text
http://127.0.0.1:25520/init.html
```

初始化页面是两步向导。

第一步：运行与路径配置。

- 运行模式，默认生产模式。
- Web 监听地址和 Web 端口。
- 数据库路径。
- Web 控制台是否启用 HTTPS。
- 服务端还需要填写控制监听地址、控制端口、数据监听地址、数据端口和公网端口范围。
- 客户端只需要填写默认控制端口、默认数据端口，不需要填写默认服务端地址。

第二步：管理员账号和协议确认。

- 控制台账号不能为空。
- 控制台密码至少 8 位，必须同时包含字母和数字。
- 必须勾选“已阅读并同意《用户协议》”。

项目不再内置默认账号 `admin` 和默认密码 `admin123456`，首次初始化必须自己创建管理员账号。

启用 Web HTTPS 时有两个选项：

- 自动生成测试证书：程序生成自签名测试证书，并写入 `ssl/web.crt`、`ssl/web.key`。
- 填写证书：在页面粘贴证书 PEM 和私钥 PEM，程序写入 `ssl/web.crt`、`ssl/web.key`。

初始化成功后会自动创建：

- `xiaoliang02_*/config/config.json`
- `xiaoliang02_*/data/`
- `xiaoliang02_*/logs/`
- `xiaoliang02_*/ssl/`
- SM2 密钥文件
- SQLite 数据库
- 管理员账号

初始化时 `auth.jwt_secret` 会随机生成，不使用固定字符串。

## 4. 启动方式

源码启动服务端：

```powershell
cd D:\2025Code\ai\natt\nattserver
go run .
```

源码启动客户端：

```powershell
cd D:\2025Code\ai\natt\nattuser
go run .
```

显式指定配置：

```powershell
cd D:\2025Code\ai\natt\nattserver
go run . -config xiaoliang02_server\config\config.json

cd D:\2025Code\ai\natt\nattuser
go run . -config xiaoliang02_user\config\config.json
```

编译：

```powershell
cd D:\2025Code\ai\natt\nattserver
go build ./...

cd D:\2025Code\ai\natt\nattuser
go build ./...
```

把编译后的 exe 放到空文件夹后直接启动，程序会以 exe 所在目录作为工作目录，并在该目录下创建 `xiaoliang02_server/` 或 `xiaoliang02_user/`。

## 5. 登录说明

登录页面需要：

- 输入初始化时创建的管理员账号和密码。
- 查看图片验证码并输入验证码。
- 勾选“已阅读并同意《用户协议》”。

生产模式下，前端会先获取 SM2 公钥，再把密码加密后提交。验证码接口只返回 `captcha_id` 和图片地址，不会把算式文本写到 HTML 或 DOM 中。

同一个 IP 登录错误累计 10 次会触发封禁，封禁时长递进为：

```text
5分钟 -> 10分钟 -> 30分钟 -> 1小时 -> 6小时
```

该限制保存在内存中，重启 exe 后重置。

## 6. 服务端使用流程

1. 启动并初始化 `nattserver`。
2. 登录服务端 Web 后台。
3. 在“客户端授权”中新增客户端，复制生成的客户端秘钥。
4. 在“隧道”中新增 TCP 隧道。
5. 填写隧道名称、公网监听地址、公网监听端口。
6. 根据需要勾选“nattuser 连接后自动启动”。
7. 复制生成的隧道秘钥，交给客户端配置。
8. 隧道启动后，服务端表格会显示状态、状态详情、上行字节、下行字节、秘钥摘要和操作按钮。

注意：

- 服务端隧道不填写内网本地地址和本地端口。
- 公网监听端口创建时必须是 `1-65535`。
- `remote_port_min` 可以为 `0`，但创建隧道时不能把公网监听端口设为 `0`。
- 如果客户端未在线时点击“启动”，服务端隧道会进入“待连接”状态并开启自动启动；客户端重新上线后会自动启动为运行中。
- 服务端停止隧道后，会通知客户端同步状态。

## 7. 客户端使用流程

1. 启动并初始化 `nattuser`。
2. 登录客户端 Web 后台。
3. 在“隧道连接”中新增连接。
4. 服务端地址默认空白，需要用户手动填写。
5. 控制端口和数据端口默认读取 `config.json` 中的 `server_defaults.control_port`、`server_defaults.data_port`。
6. 填写服务端提供的隧道秘钥。
7. 填写本地目标地址和本地目标端口，例如 `127.0.0.1:3306`。
8. 保存后启动连接。

连接成功后，客户端表格会显示：

- 服务端地址。
- 控制端口。
- 数据端口。
- 服务端秘钥摘要和查看按钮。
- 本地目标。
- 服务端公网监听端口。
- 状态和状态详情。

如果服务端隧道处于停止状态，客户端状态详情会显示：

```text
服务端暂停了隧道连接，请通知服务端人员启动隧道。
```

如果同一个服务端隧道已被其他客户端占用，客户端状态详情会显示：

```text
该连接正在占用，不得连接
```

## 8. 隧道规则

本项目采用“服务端管公网，客户端管本地”的规则：

- `nattserver` 管理公网监听地址、端口、状态、秘钥和流量统计。
- `nattuser` 管理服务端地址、控制端口、数据端口、服务端秘钥、本地地址和本地端口。
- 一个服务端隧道同一时间只允许一个客户端连接占用。
- 客户端因网络异常或进程短暂中断掉线后，只要该连接仍允许自启动，会自动重连。
- 在客户端后台点击“断开”会把该连接改为停止状态并关闭自启动；之后不会自动重连，必须再次点击“连接”才会重新允许自动重连。
- 服务端隧道因控制连接离线进入 `error`，且状态详情为“隧道控制连接已离线”时，原客户端重新上线或发送心跳后会自动恢复隧道；端口占用、配置错误等其他异常不会被自动清除。
- 客户端心跳会读取服务端返回的隧道状态；即使错过 `tunnel_stop` 指令，后续心跳也会把本地连接同步为服务端 `stopped` 或 `error` 的状态详情。
- 服务端停止或删除隧道后，公网端口会释放。
- 状态详情过长时，表格只显示前 20 个字符，可点击图标查看完整内容。

## 9. 配置页

两端配置页分为：

- 可热更新配置：允许直接修改保存。
- 当前配置：只读展示，不允许页面热更新。

不可热更新配置需要修改 `config.json` 后重启程序。

MCP Key 不允许前端自定义。启用 MCP 时系统会自动生成 Key，后台可通过“查看”按钮查看完整 Key，也可以重置 Key。

## 10. 审计日志

审计日志文件位置：

```text
nattserver\xiaoliang02_server\logs\audit\YYYY-MM-DD.jsonl
nattuser\xiaoliang02_user\logs\audit\YYYY-MM-DD.jsonl
```

后台审计页从 JSONL 文件分页读取。审计内容包括：

- 登录成功和登录失败。
- 隧道创建、启动、停止、删除。
- MCP Key 查看和轮换。
- MCP `tools/list` 和 `tools/call`。
- 客户端连接相关操作。

普通日志位于：

```text
nattserver\xiaoliang02_server\logs\YYYY-MM-DD.log
nattuser\xiaoliang02_user\logs\YYYY-MM-DD.log
```

普通日志输出包含触发的 Go 文件名和行号。

## 11. MCP 使用

两端 MCP 都使用标准 Streamable HTTP JSON-RPC，地址为：

```text
POST http://<host>:<port>/mcp
```

鉴权方式二选一：

```text
Authorization: Bearer <MCP_KEY>
X-MCP-Token: <MCP_KEY>
```

初始化示例：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-11-25",
    "capabilities": {},
    "clientInfo": {
      "name": "codex",
      "version": "1.0.0"
    }
  }
}
```

工具发现示例：

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/list",
  "params": {}
}
```

工具调用示例：

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "server.list_tunnels",
    "arguments": {
      "page": 1,
      "page_size": 20
    }
  }
}
```

服务端可用工具：

- `server.list_clients`
- `server.get_client`
- `server.list_tunnels`
- `server.create_tunnel`：创建服务端 TCP 隧道；省略 `auto_start` 时默认 `true`，创建后进入待连接状态。
- `server.start_tunnel`
- `server.stop_tunnel`
- `server.delete_tunnel`
- `server.get_dashboard`

客户端可用工具：

- `client.list_tunnel_connections`
- `client.list_servers`
- `client.create_tunnel_connection`：新增客户端隧道连接。
- `client.delete_tunnel_connection`：删除客户端隧道连接。
- `client.connect_tunnel`
- `client.connect_server`
- `client.disconnect_tunnel`
- `client.disconnect_server`
- `client.list_tunnels`
- `client.get_network_status`

`client.create_tunnel_connection` 参数说明：

- 必填：`name`、`server_host`、`client_secret`、`local_host`、`local_port`。
- 可选：`server_port`、`data_port`、`auto_start`、`remark`。
- `server_port` 或 `data_port` 省略或为 `0` 时，使用客户端 `config.json` 中的 `server_defaults.control_port` 和 `server_defaults.data_port`。

新增客户端隧道连接示例：

```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "tools/call",
  "params": {
    "name": "client.create_tunnel_connection",
    "arguments": {
      "name": "办公内网服务",
      "server_host": "1.2.3.4",
      "client_secret": "xiaoliang_xxx",
      "local_host": "127.0.0.1",
      "local_port": 8080,
      "auto_start": true
    }
  }
}
```

可以给 Codex 这样的提示词：

```text
连接 http://127.0.0.1:25510/mcp，使用 Bearer Token，列出所有服务端隧道并告诉我哪些处于 error 状态。
```

```text
连接 http://127.0.0.1:25520/mcp，查询客户端隧道连接状态，如果有错误请总结状态详情。
```

```text
连接 http://127.0.0.1:25520/mcp，新增一个到 1.2.3.4 的隧道连接，本地目标是 127.0.0.1:8080，控制端口和数据端口使用默认配置。
```

## 12. 常见问题

启动后提示端口被占用：

- 检查对应端口是否被其他程序占用。
- 服务端重点检查 `25510`、`25511`、`25512`。
- 客户端重点检查 `25520`。

空文件夹启动后不知道访问哪里：

- 服务端访问 `http://127.0.0.1:25510/init.html`。
- 客户端访问 `http://127.0.0.1:25520/init.html`。

生产模式登录提示加密密码不正确：

- 确认登录页已加载 `sm2.js`。
- 确认 `/api/*/v1/auth/sm2-public-key` 返回 `public_key_hex`。
- 确认没有直接用接口提交明文密码。

客户端连接失败：

- 检查服务端地址、控制端口、数据端口。
- 检查服务端隧道秘钥是否正确。
- 检查服务端隧道是否为 `running` 或允许客户端连接后自动启动。
- 检查该服务端隧道是否已被其他客户端占用。
- 检查本地目标地址和端口是否能在客户端机器访问。

MCP 无法调用：

- 确认后台已启用 MCP。
- 确认使用的是 `POST /mcp`。
- 确认请求头包含正确的 MCP Key。
- 确认没有继续调用旧路径。
