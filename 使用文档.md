# NATT 内网穿透使用文档

本文说明如何启动、配置和使用 `nattserver` 服务端与 `nattuser` 客户端。当前版本只支持 TCP 端口穿透，通信模型为“控制连接 + 数据连接分离”。

## 1. 项目目录

```text
D:\2025Code\ai\natt
  nattserver\    服务端
  nattuser\      客户端
  commonGin\     参考项目
```

服务端默认文件：

```text
nattserver\config\config.json
nattserver\data\nattserver.db
nattserver\logs\YYYY-MM-DD.log
nattserver\logs\audit\YYYY-MM-DD.jsonl
```

客户端默认文件：

```text
nattuser\config\config.json
nattuser\data\nattuser.db
nattuser\logs\YYYY-MM-DD.log
nattuser\logs\audit\YYYY-MM-DD.jsonl
```

## 2. 默认端口

服务端：

| 用途 | 默认值 |
| --- | --- |
| Web 管理后台 | `0.0.0.0:8080` |
| MCP 接口 | `0.0.0.0:8080/mcp/*` |
| 客户端控制连接 | `0.0.0.0:7000` |
| 客户端数据连接 | `0.0.0.0:7001` |
| 公网映射端口范围 | `10000-60000` |

客户端：

| 用途 | 默认值 |
| --- | --- |
| 本地 Web 管理后台 | `127.0.0.1:18080` |
| MCP 接口 | `127.0.0.1:18080/mcp/*` |

默认管理员账号：

| 项目 | 用户名 | 密码 |
| --- | --- | --- |
| 服务端 | `admin` | `admin123456` |
| 客户端 | `admin` | `admin123456` |

生产环境建议在首次启动前设置管理员环境变量，数据库已经初始化后再改环境变量不会自动修改旧用户密码。

```powershell
$env:NATT_SERVER_ADMIN_USERNAME="admin"
$env:NATT_SERVER_ADMIN_PASSWORD="change-this-server-password"
$env:NATT_USER_ADMIN_USERNAME="admin"
$env:NATT_USER_ADMIN_PASSWORD="change-this-client-password"
```

## 3. 启动

启动服务端：

```powershell
cd D:\2025Code\ai\natt\nattserver
go run . -config config/config.json
```

访问：

```text
http://127.0.0.1:8080/login.html
```

启动客户端：

```powershell
cd D:\2025Code\ai\natt\nattuser
go run . -config config/config.json
```

访问：

```text
http://127.0.0.1:18080/login.html
```

前端现在是多页面结构：`login.html`、`dashboard.html`、`clients.html` 或 `servers.html`、`tunnels.html`、`config.html`、`audit.html`。未登录访问模块页会跳回登录页。

## 4. 基本流程

### 4.1 服务端创建客户端授权

1. 打开服务端后台：`http://127.0.0.1:8080/login.html`
2. 登录后进入“客户端”页面。
3. 新增客户端，保存后记录弹出的客户端秘钥。
4. 客户端秘钥只显示一次，服务端数据库只保存哈希和摘要。

### 4.2 客户端连接服务端

1. 打开客户端后台：`http://127.0.0.1:18080/login.html`
2. 进入“服务端”页面并新增服务端连接。
3. 填写服务端地址、控制端口 `7000`、数据端口 `7001`、客户端秘钥和 TLS 开关。
4. 保存后点击连接，状态变为 `connected` 即可。

### 4.3 服务端创建公网隧道

服务端隧道只管理公网监听和客户端归属，不再填写内网本地地址/端口。

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 名称 | `ssh-home` | 隧道名称 |
| 客户端 | `client-a` | 已授权并在线的客户端 |
| 公网监听地址 | `0.0.0.0` | 服务端监听地址 |
| 公网监听端口 | `10022` | 外部访问端口，必须在允许范围内且不能冲突 |
| 自启动 | 按需 | 客户端重连后自动恢复 |

保存后记下服务端返回的隧道 ID。

### 4.4 客户端绑定本地目标

客户端进入“隧道绑定”页面，新增本地绑定：

| 字段 | 示例 | 说明 |
| --- | --- | --- |
| 服务端连接 | `server-a` | 对应 `server_connections.id` |
| 服务端隧道 ID | `1` | 服务端创建的隧道 ID |
| 本地地址 | `127.0.0.1` | 客户端机器可访问的内网服务地址 |
| 本地端口 | `22` | 客户端机器上的内网服务端口 |
| 启用 | `是` | 未配置或禁用时会向服务端返回 `data_close` 错误 |

绑定完成后，在服务端启动隧道。外部访问：

```text
服务端公网IP:公网监听端口
```

会被转发到：

```text
客户端本地地址:客户端本地端口
```

示例：

```text
1.2.3.4:10022 -> 客户端 127.0.0.1:22
```

## 5. 配置

默认配置格式是 JSON：

```text
config/config.json
```

显式传入 YAML 仍然兼容：

```powershell
go run . -config config/config.yaml
```

不传 `-config` 时会优先读取 `config/config.json`，不存在时回退 `config/config.yaml`。JSON 标准不支持注释，配置说明请写在部署文档里。

MCP 配置只保留启停和访问令牌：

```json
{
  "mcp": {
    "enabled": true,
    "access_token": "replace-with-token"
  }
}
```

旧配置里的 `mcp.host`、`mcp.port` 可以被解析，但不再生效。MCP 总是挂在当前 HTTP 服务端口下。

## 6. MCP 调用

服务端 MCP：

```text
POST http://127.0.0.1:8080/mcp/tools/call
Authorization: Bearer replace-with-server-mcp-token
Content-Type: application/json
```

客户端 MCP：

```text
POST http://127.0.0.1:18080/mcp/tools/call
Authorization: Bearer replace-with-client-mcp-token
Content-Type: application/json
```

请求示例：

```json
{
  "tool": "server.list_clients",
  "params": {
    "page": 1,
    "page_size": 20
  }
}
```

服务端创建隧道示例，不包含 `local_host/local_port`：

```json
{
  "tool": "server.create_tunnel",
  "params": {
    "name": "ssh-home",
    "client_id": 1,
    "protocol": "tcp",
    "remote_host": "0.0.0.0",
    "remote_port": 10022,
    "auto_start": true,
    "remark": "ssh tunnel"
  }
}
```

MCP 返回格式：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

## 7. 审计与日志

普通运行日志：

```text
nattserver\logs\YYYY-MM-DD.log
nattuser\logs\YYYY-MM-DD.log
```

普通日志会包含触发位置，例如：

```text
[INFO] routes.go:61 server routes registered
```

审计日志已经从 SQLite 迁移为 JSONL 文件：

```text
nattserver\logs\audit\YYYY-MM-DD.jsonl
nattuser\logs\audit\YYYY-MM-DD.jsonl
```

首次启动时会将旧 SQLite `audit_logs` 导出到 JSONL，并写入迁移标记避免重复导出。Web 审计页面和 `/audit-logs` API 读取 JSONL 文件分页展示。

## 10. 常见问题

隧道启动失败：

- 确认客户端在线。
- 确认 `remote_port` 在允许范围内且未被占用。
- 确认客户端已为对应 `server_connection_id + server_tunnel_id` 创建本地绑定。
- 确认本地绑定启用，且本地服务可从客户端机器访问。
- 确认服务端防火墙放行公网映射端口，客户端可访问服务端数据端口 `7001`。

MCP 返回 unauthorized：

- 确认 `mcp.enabled=true`。
- 确认 `mcp.access_token` 已配置。
- 确认请求头为 `Authorization: Bearer <token>`。
- 确认调用地址是 HTTP 端口下的 `/mcp/tools/call`，不是旧的独立 MCP 端口。
